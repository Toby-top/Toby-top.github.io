Fitting | 学习、刷题与过拟合

January 27, 2026·1285 字·3 分钟
对历史精确或必然的解释，通常不是伟大或正确，而是穿凿附会的谬误.
— Ich. Selbst. tobyTop.

Intro.
#

我曾经在备考时刷过大量的题，但结果是成绩变得起起落落TAT.
最近在学习 Machine Learning 时了解到一个新的概念—“过拟合”
本文将引入机器学习的这一概念，尝试重新理解一个大部份情况下默认为真的前提—“刷题 = 学习”

Main Body.
#

1. 什么是过拟合
#

In mathematical modeling, overfitting is the production of an analysis that corresponds too closely or exactly to a particular set of data, and may therefore fail to fit to additional data or predict future observations reliably.
— Wikipedia

一个模型在训练集上的表现越好，并不意味其越“聪明”。当一个模型对训练集数据的每一个偶然，噪声，甚至编号都全部记住且产生依赖时，这个模型的在训练集中所得的分数会越来愈高，但与此同时，在测试集，或者说现实世界中的表现却迅速崩塌。

下面这个散点图记录了一个城市一年里每个月某一天的平均气温，我们的目的是利用最小二乘法来找一条可以用于预测气温的曲线。

可以看见，一开始图像确实是越来越精确的，但随着多项式次数的增加，这条的曲线却变得越来越诡异，最终到11次时扭曲不堪。此时曲线完美地经过了散点图上的每一个点，但是离开了这些点的数据变得十分诡异，这个曲线已完全失去预测意义。

2. 将训练集换为题库
#

让我们作一个不严谨的简单类比：
训练集 (training set) ≈ 做过的题目
测试集 (test set) ≈ 没见过的新题
模型性能 (performance) ≈ 成绩或解题成功率

刷题就相当于向模型不断投喂新数据，当刷题的量达到一定程度，变化便悄然发生。当学习者开始依赖“题型模版”； “常见套路”；“关键词触发的固定解法”时，刷题所优化的点就不是“理解问题结构”了，而是“在已知分布内尽快命中答案”。

这时，学习就会向过拟合滑落。

3. “记忆”与“理解”的边界
#

学而不思则过拟合，思而不学则欠拟合.

本文并不是否定“做题”的价值，而是在强调反馈回路的方向。

当学习主要依赖“是否做对”作为唯一反馈时，大脑自然会选择成本最低的策略：
与其抽象(abstract)出稳定的结构，不如直接记住出现过的形式。

于是，“理解”逐渐被替换成一种更隐蔽的能力——
对题目分布的熟悉程度。

这种能力在题库稳定、风格一致时表现良好，
却在题目稍有变化时迅速失效。
它更像是一种缓存，而不是模型。

4. 引入“正则化”
#

在机器学习中，防止过拟合的核心思路并非“少训练”，
而是限制模型将注意力用在细节上。

这些方法被统称为正则化(Regularisation)，
它们的共同点是：
人为提高“记住训练集”的成本。

若将这一思想迁移到学习中，则可转述为以下技巧：

  • 更换题源、刻意避开熟悉的出题风格
  • 在解题前先写出思路，而不是直接计算
  • 对一道已做对的题，解释“如果条件改变会发生什么”
  • 用语言而不是步骤复述解法

这些做法并不会立刻提高分数，
却能显著提升面对新问题时的稳定性。

从这个角度看，
刷题并无问题，
应避免的是，无约束地刷题。

Concl.
#

若过拟合意味着“学得太具体”，
那么另一端的欠拟合，是否就值得向往？

答案可能并不乐观。

欠拟合的模型看似“泛化”，
实则只是什么都没真正抓住。

对应到学习中，
逃避刷题、拒绝训练、只停留在模糊理解，
同样无法在真实问题中给出可靠输出。

应在“理解”与“记忆”间，持续地调整那条看不见的边界。
